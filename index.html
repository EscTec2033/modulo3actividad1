Investiga y Desarrolla.

1. ¿Qué es un Dato?

Es la representación de una variable que puede ser cuantitativa o cualitativa. Se expresa a través de una secuencia de símbolos, números o letras, y describe hechos empíricos.

2. ¿Qué es información?

Es un conjunto organizado de datos relevantes para uno o más sujetos, que extraen de él conocimiento1. En otras palabras, es una serie de conocimientos comunicados, compartidos o transmitidos, constituyendo algún tipo de mensaje.

3. Diferencia entre dato e información.

Los datos son hechos y cifras crudas, sin procesar. Por otro lado, la información es el resultado del procesamiento y análisis de los datos. En resumen, los datos por sí solos no tienen significado. Es a través del proceso de transformar los datos en información que podemos obtener conocimiento y sacar conclusiones.

4. Tipos de datos.

En una base de datos, los tipos de datos determinan el tipo de información que se puede almacenar en un campo o columna. Aquí tienes algunos de los tipos de datos más comunes:


  • Datos numéricos: Estos tipos se utilizan para almacenar números. Ejemplos incluyen enteros (como INT), números decimales (como DECIMAL o FLOAT), y valores monetarios (como MONEY).

  • Datos de texto: Se utilizan para almacenar texto o cadenas de caracteres. Algunos ejemplos son VARCHAR (para texto variable) y CHAR (para texto fijo).

  • Datos booleanos: Estos almacenan valores lógicos, como TRUE o FALSE. En SQL, se suelen representar como BIT.

  • Datos de fecha y hora: Se utilizan para almacenar información de tiempo. Ejemplos incluyen DATE, TIME, y DATETIME.

  • Datos binarios: Estos tipos almacenan datos en formato binario, como imágenes o archivos. Ejemplos son BLOB o VARBINARY.

5. ¿Qué es un modelado de Datos?

Es un proceso fundamental en el desarrollo de sistemas de información. Consiste en crear una representación estructurada y organizada de los datos de una organización, con el objetivo de mejorar la comunicación y la precisión en el manejo de los datos.

6. Tipos de bases de Datos, según: Variabilidad, Contenido, Modelo de base de datos.

1. Variabilidad de la base de datos:


  • Bases de datos estáticas: Están diseñadas principalmente para la lectura de datos. Se utilizan para almacenar y registrar información histórica, lo que permite realizar estudios y proyecciones estadísticas.

  • Bases de datos dinámicas: Estas bases de datos permiten tanto la lectura como la escritura de datos. Son ideales para aplicaciones en tiempo real y sistemas interactivos.

2. Contenido.


  • Bases de datos bibliográficas: Estos sistemas registran y clasifican varios campos de datos conjuntamente o por separado. Por ejemplo, datos relativos a la información de un libro, como título y autor.

  • Bases de datos de texto completo: Almacenan y permiten la búsqueda de texto completo en documentos o registros.

3. Modelo de base de datos.


  • Bases de datos jerárquicas: Organizan los datos en forma de árbol invertido o de raíz.

  • Bases de datos de red: Tienen un nodo central con varios nodos padres, a diferencia de la clasificación jerárquica.

7. ¿Qué es el modelo relacional?

Es un método para estructurar datos utilizando relaciones, mediante estructuras en forma de cuadrícula que consisten en columnas y filas. Fue propuesto por Edgar F. Codd en 1969 y se ha convertido en el modelo dominante para aplicaciones comerciales.

8. Describa los elementos del Diagrama entidad Relacional.

Este tipo de diagrama se utiliza en el diseño de bases de datos para visualizar las entidades relevantes en un sistema, sus atributos y las relaciones entre ellas.

9. ¿Qué es una entidad?

Representan objetos del mundo real o conceptual que tienen existencia independiente y pueden ser identificados. Se representan mediante rectángulos y se nombran de manera descriptiva, reflejando aspectos clave del dominio del problema que se está abordando.

10. ¿Qué es un atributo y sus tipos?

Son características que describen y definen una entidad, aportando detalles o información específica sobre ella. Estos atributos se muestran dentro de los rectángulos que representan a las entidades y se identifican con etiquetas. Algunos de sus tipos son:


  • Atributos simples: Contienen un solo valor o dato.

  • Atributos compuestos: Almacenan varios valores de datos, como los atributos de la dirección postal.

  • Atributos multi-valuados: Almacenan múltiples valores para los mismos atributos. Por ejemplo, una base de datos de inventario puede tener varios atributos para la cantidad de productos en stock, como “cantidad en almacén”, “cantidad en camión” y "cantidad en el mostrador".

11. ¿Qué es una relación?

Indican la asociación y conexión entre diferentes entidades. Se representan con líneas que unen las entidades relacionadas y se etiquetan para describir la naturaleza de la conexión (por ejemplo, “uno a uno”, “uno a muchos” o “muchos a muchos”).

12. ¿Qué son las llaves primarias, foránea y sus características?

Una llave primaria es un atributo o conjunto de atributos que identifican de forma única una fila en una tabla12. Por ejemplo, podríamos tener una tabla de clientes donde el número de identificación de cada cliente se utiliza como llave primaria, garantizando la unicidad de cada registro. Por otro lado, una llave foránea es un atributo o conjunto de atributos en una tabla que hacen referencia a la llave primaria de otra tabla. En otras palabras, las llaves foráneas permiten relacionar tablas dentro de una base de datos1. Por ejemplo, si tenemos una tabla de pedidos, podríamos tener una llave foránea que se relaciona con la llave primaria de la tabla de clientes para indicar qué cliente realizó cada pedido.

13. ¿Qué son las reglas del Negocio?

Una regla de negocio es una condición que debe satisfacerse cuando se realiza una actividad empresarial. Estas reglas pueden imponer políticas de negocio, tomar decisiones o inferir nuevos datos a partir de los datos existentes. Por ejemplo, en SQL Server Master Data Services, las reglas de negocio son instrucciones If/Then/Else que se crean en Master Data Manager. Si un valor de atributo cumple una condición especificada, se realiza una acción; de lo contrario, se ejecuta una acción alternativa. Estas acciones pueden incluir establecer un valor predeterminado o cambiar un valor. En resumen, las reglas de negocio guían la toma de decisiones diarias dentro de las empresas, trazando relaciones entre objetos como nombres de clientes y sus pedidos correspondientes.

14. ¿Qué es la cardinalidad o mapeo?

Es la relación entre dos o más entidades en un sistema de información. Específicamente, indica la cantidad de conjuntos de registros o filas que están relacionados entre sí en una relación de muchos-a-muchos o muchos-a-uno.
